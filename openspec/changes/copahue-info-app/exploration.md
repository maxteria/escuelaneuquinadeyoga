## Context
The `copahue-info-app` change gathers everything that can be said confidently about the Caviahue-Copahue volcanic-thermal complex from Argentine authorities. The goal is to store static background data once (geology, location, the thermal complex description, overnight contact/operating hours) and layer it with live volcanic/risk/visitor signals (activity bulletins, ash alerts, seismic tables, cameras, thermal status notices). All sources are official Argentine entities (SEGEMAR/OAVV, SMN/VAAC, Termas del Neuquén, INPRES), so the app can present the information without leaving the authorized domain.

## Sources
- [SEGEMAR – Observatorio Argentino de Vigilancia Volcánica (OAVV), Copahue page](https://oavv.segemar.gob.ar/volcanes/copahue): volcano description, risk ranking, monitoring network, live-report placeholders (activity bulletin, ash forecast, cameras, sismogramas) and attribution to the VAAC Buenos Aires/SMN ash forecasts.
- [Termas del Neuquén – Termas de Copahue](https://termas.neuquen.gov.ar/termas-de-copahue/): thermal complex overview, mineralogy, recommendations, contact phones, daily hours and the biannual summer season.
- [Termas del Neuquén news item “El paraíso termal abrió sus puertas…” (15 Jul 2025)](https://termas.neuquen.gov.ar/2025/07/15/): example season status bulletin that confirms the Copahue camping is open through 30 September and highlights timing-sensitive messaging for visitors.
- [Termas del Neuquén RSS feed](https://termas.neuquen.gov.ar/feed/): canonical, hourly-renewed stream of news/avisos that can be polled to detect new maintenance, access or closure bulletins.
- [Servicio Meteorológico Nacional (SMN)](https://www.argentina.gob.ar/servicio-meteorologico-nacional): Argentinian authority hosting the VAAC Buenos Aires responsible for ash forecasts cited on the OAVV page (6/12/18-hour products).
- [INPRES “Últimos sismos determinados”](http://contenidos.inpres.gob.ar/sismologia/xultimos): national seismic table listing the latest ~50 events with location, magnitude, depth and update time.

## Data classification

### Evergreen data (store once)
1. **Volcano profile and hazard context** – The OAVV Copahue page provides the province, coordinates, elevation (2,937 m), risk ranking (=1) and the historical description of the Caviahue-Copahue complex (stratovolcano, active east crater, eruptive history, fumarolic lake, nearby settlements). Treat these facts as “evergreen”; refresh only when SEGEMAR publishes a revised profile or ranking. Source: [OAVV Copahue](https://oavv.segemar.gob.ar/volcanes/copahue).
2. **Thermal complex description & visitor guidance** – Termas del Neuquén explains the mineral wealth, therapy capabilities, recommended clothing/health hygiene, contact channels, and the default open hours (8‑20 daily) plus typical summer season (1 Dec–30 Apr). Record this narrative and contact block as static content until province-issued policies change. Source: [Termas de Copahue](https://termas.neuquen.gov.ar/termas-de-copahue/).
3. **Seasonal anchor points** – The July 2025 news item confirms the “Termas Nieve” winter product runs until 30 September, showing that the government publishes date-bound notices when they open/close the resort. Capture the baseline dates but consult the news feed before each season. Source: [Termas news 15 Jul 2025](https://termas.neuquen.gov.ar/2025/07/15/).

### Dynamic data (live feeds ranked by priority)
1. **SEGEMAR/OAVV volcanic monitoring bulletin** *(Priority 1)* – The “Último reporte de actividad volcánica”, ash prognosis panel and the event evolution graphs on the Copahue page are treated as continuous bulletins. There is no public API, so plan to fetch the HTML page, extract the timestamped bulletin text/values, and cache the parsed version for ~5‑10 min TTL before re-fetching. If SEGEMAR offers a direct feed later, swap the scraper for their official endpoint. Always tag the data as “Live SEGEMAR report” and surface the source timestamp in the UI to avoid misinformation. Source: [OAVV Copahue](https://oavv.segemar.gob.ar/volcanes/copahue).
2. **VAAC Buenos Aires ash forecasts (SMN)** *(Priority 2)* – OAVV explicitly references the VAAC Buenos Aires/SMN ash forecasts for 6/12/18-hour projections. There is no Arg-specific REST API, so treat the SMN product as a published bulletin (PDF/HTML) that can be polled roughly every 6 hours (or tied to VAAC release timestamps). Show the VAAC/SMN attribution and keep the forecast’s valid-until metadata; fallback to SEGEMAR’s summarised ash map when SMN content fails. Source: [OAVV Copahue](https://oavv.segemar.gob.ar/volcanes/copahue) + [SMN](https://www.argentina.gob.ar/servicio-meteorologico-nacional).
3. **INPRES seismic list** *(Priority 3)* – The “Últimos sismos determinados” table publishes the latest earthquakes (magnitude, depth, lat/long, origin time). The page regenerates automatically when new events arrive, so set up a parser for the HTML table (or request CSV if available) and poll every 5‑15 minutes. Display magnitude, location, and local time plus a link back to INPRES for full reports. Source: [INPRES sismología](http://contenidos.inpres.gob.ar/sismologia/xultimos).
4. **Thermal complex status & advisories** *(Priority 4)* – The Termas del Neuquén RSS feed (hourly) captures new notices about operating days, maintenance, recruitment/competitions, or access conditions. Subscribe to `https://termas.neuquen.gov.ar/feed/`, parse the latest posts, and treat any Copahue/Caviahue mention as a trigger to update the “operational status” card. Use caching (1‑hour TTL) and cross-validate that a story references Copahue before surfacing it. Sources: [Termas RSS](https://termas.neuquen.gov.ar/feed/) and [Termas news 15 Jul 2025](https://termas.neuquen.gov.ar/2025/07/15/).
5. **Monitoring cameras & timelapse images** *(Priority 5)* – The OAVV page exposes timelapse widgets for Agrio Superior, Laguna Escondida, Nodo Aeródromo and Estación Mellizas, along with the note “Imágenes provenientes de las cámaras de monitoreo de SEGEMAR-OAVV y de SERNAGEOMIN-OVDAS”. There is no documented API, so the fallback is to capture the image URLs (or embed the iframes) with a strict rate limit (e.g., one fetch per camera per minute) and cache the latest frame for 30‑60 s to avoid stressing the servers. Ask SEGEMAR for permission/technical guidance before redistributing (SERNAGEOMIN-era frames also carry cross-border conditions). Source: [OAVV Copahue](https://oavv.segemar.gob.ar/volcanes/copahue).
6. **Visitor risk guidance & disclaimers** *(Priority 6)* – Combine the evergreen recommendations from Termas (clothing, health, mountain hazards) with OAVV’s risk narrative (proximity of Caviahue/Copahue, ranking =1). Mark this combination as “stable guidance” but refresh the messaging when OAVV publishes escalated alerts. Always cite the original text to ensure no stray advice is shown. Sources: [Termas de Copahue](https://termas.neuquen.gov.ar/termas-de-copahue/) and [OAVV Copahue](https://oavv.segemar.gob.ar/volcanes/copahue).

### Integration considerations
- **Refresh cadence**: Poll SEGEMAR & INPRES every 5‑15 min (fastest after a bulletin or seismic spike); check SMN/VAAC every 6 hours; pull the Termas RSS hourly; fetch cameras no more than once per minute. Use server-side schedulers with jitter to avoid hammering remote hosts.
- **Caching strategy**: Persist the last successful payload for each dynamic feed, including `lastUpdated` and the raw HTML/description. If a fetch fails, surface the cached data with an “old data” banner. Evergreen data stays in a local store (database or JSON) and is only refreshed when a new government bulletin (new ranking or schedule) arrives.
- **Offline support**: Ship the static descriptions and contact info inside the app bundle so the UI still shows context when signal is absent, but mark it clearly as “reference content” versus “live data.”
- **Risk of misinformation**: Always display the source name and timestamp next to dynamic data; never extrapolate beyond what the official bulletin states. If multiple sources disagree (e.g., OAVV versus INPRES), present both with their timestamps.
- **Labeling evergreen vs dynamic**: Tag each card in the UI with metadata (e.g., `type:evergreen` or `type:live`) so visitors can distinguish the stored background from the live feed, and surface a “Refresh” button/counter tied to the TTL for each dynamic block.

## Risks/Constraints
- **No public APIs for SEGEMAR bulletins or cameras** – the Copahue page is HTML-only, so we must either screen-scrape or formalize a data-sharing agreement with SEGEMAR; without explicit permission, heavy polling may be blocked.
- **Camera feeds combine Argentinian and Chilean (SERNAGEOMIN) gear** – even though the imagery is public on OAVV, redistributing it can violate cross-border agreements; confirm licensing/support levels before embedding live streams.
- **Termas updates are delivered via a WordPress blog** – the feed is hourly but not granular (the event date is the post date), so status changes may lag; pair RSS polling with manual review of high-impact posts.
- **Risk of stale information in remote coverage** – Copahue sits in the Andes with spotty mobile coverage; cached data must be explicit about when it was captured to avoid decisions based on outdated ash or thermal forecasts.
